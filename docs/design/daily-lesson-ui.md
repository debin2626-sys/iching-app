# 日课 UI 设计方案

> 基于 PRD v3 + 现有国风宣纸设计系统
> 最后更新: 2026-04-21 v2（合并郭郭评审反馈）

## 设计原则

1. **延续现有美学** — 宣纸底色 `#f7f3eb`、金色 `#b8924a` 主调、霞光渐变、LXGW WenKai 字体
2. **内容为王** — 日课是阅读体验，不是交互体验。排版优先，动效克制
3. **移动端优先** — 80%+ 用户来自手机，所有设计先画 375px 宽度
4. **飞轮意识** — 每个页面都有明确的下一步：订阅、分享、或深入阅读
5. **主站零价格铁律** — 51yijing.com 不出现任何 ¥ 符号或价格数字，付费交易全部跳转小报童等第三方平台

### 设计细节

#### 日期区域
- 公历大字 `text-2xl`，金色 `var(--color-gold)`
- 农历 + 节气 + 干支 `text-sm`，`var(--theme-text-secondary)`
- 使用 `lunar-javascript` 库计算
- 节气名如有，加金色高亮

#### Tab 切换
- 两个 tab：「今日一卦」「每日静心」
- 复用 NavBar 的 `layoutId` 下划线动画（framer-motion）
- Active tab 金色下划线 2px，文字 `var(--color-gold)`
- Inactive tab 文字 `var(--theme-text-muted)`

#### 内容卡片
- 复用现有 `Card` 组件 variant=`default`，padding=`lg`
- 标题区：天数 + 卦名 `text-xl font-display`，金色；右侧显示「约 2 分钟」阅读时间估算（`text-xs text-muted`）
- 副标题（爻辞）`text-lg`，引号包裹，`var(--theme-text-secondary)`
- 分隔线：复用 `.divider-gold`（金色渐变线）
- 各段落标题用 emoji + 文字，`text-base font-semibold`
- 正文 `text-sm leading-relaxed`，`var(--theme-text-primary)`
- 吉凶提醒区（根据 Hexagram.nature 字段触发）：
  - `nature=ji`（吉）：「⚠️ 过盛则衰」提醒，左边框 `border-l-2 border-gold`，浅金底色 `rgba(184,146,74,0.08)`
  - `nature=xiong`（凶）：「🛡️ 化险为夷」提醒，左边框 `border-l-2 border-red-400`，浅红底色 `rgba(220,38,38,0.06)`，文案侧重"如何避险、转化"
  - `nature=ji_xiong_hun`（吉凶混合）：「⚖️ 吉凶参半」提醒，左边框 `border-l-2 border-amber-500`，浅琥珀底色 `rgba(245,158,11,0.06)`，文案侧重"把握分寸、趋吉避凶"
  - `nature=ping`（平）：不显示提醒区

#### 道家清静派 Tab 内容
- 结构类似但替换为：经典原文 → 静心引导（2分钟）→ 一日一悟
- 静心引导区域：特殊样式，浅色底 + 圆角，呼吸节奏用 `···` 分隔
- 冥想词用斜体 `italic`，营造沉浸感

#### 前后天导航
- 左右箭头 + 中间太极图标 ☯
- `/daily` 今日页：按日历顺序，「← 前一天 ☯ 后一天 →」
- `/daily/yijing/[slug]` 永久内容页：按 dayIndex 序列，「← 上一条 ☯ 下一条 →」
- 到达第一条/最后一条时箭头变灰 `opacity-30 pointer-events-none`

#### 操作栏（合规：主站零价格）
- 「深度决策分析 →」：`Button` variant=`primary`（朱红色），点击跳转小报童专栏 `xiaobot.net/p/51yijing/xxx`，`target="_blank"`
- 「分享今日智慧」：复用 `.btn-divine` 样式（金色边框）
- 「查看完整卦辞解读 →」：文字链接，金色
- 视觉优先级：朱红实心 > 金色描边 > 文字链接
- ⛔ 按钮文案不得出现 ¥ 符号、价格数字、「购买」「付费」等字样

#### 订阅区（双点位策略）

订阅是留存引擎核心，不能藏在第 7 屏。采用双点位：

**点位 1 — 内容卡片结尾 inline 订阅框**：
- 嵌入内容卡片底部（行动建议之后），与内容自然衔接
- 文案：「🔔 免费订阅，每天早上 8:00 收到一条古典智慧」
- 邮箱输入框 + 订阅按钮横排
- 已订阅用户：此区域变为 ✅「已订阅，明天 8:00 见！」

**点位 2 — 首次访问吸底条 sticky bottom**：
- 仅首次访问用户显示（`localStorage.dailySubscribeDismissed` 控制）
- 固定在屏幕底部，高度 ~56px
- 文案：「📬 免费订阅每日智慧」+ [订阅] 按钮 + ✕ 关闭
- 关闭后 24h 内不再显示
- 已订阅用户永不显示

**独立订阅区（页面下方，新用户可见）**：
- 独立卡片，浅金底色 `rgba(184,146,74,0.08)`
- 标题 `text-base font-semibold`
- 邮箱输入框 + 订阅按钮横排
- 输入框：`border-gold/30` 边框，focus 时 `border-gold`
- 按钮：金色实心 `bg-gold text-white`
- 底部灰字：Coming Soon 微信服务号
- 老用户（已订阅 + 访问 > 3 次）：隐藏或变「修改订阅偏好」

#### Coming Soon 流派
- 6 个小卡片，3×2 网格
- 每个卡片：图标 + 流派名 + 一行 slogan + 行内「🔔 上线通知我」按钮
- 移动端无 hover，改为默认 `opacity-80`，点击展开 slogan + 订阅按钮
- 底部 CTA：「感兴趣？留下邮箱，上线第一时间通知你」

#### 访问状态感知（新老用户分层）

根据 `localStorage.dailyVisitCount` 区分新老用户，减少老用户信息过载：

| 条件 | 显示模块 | 隐藏/折叠模块 |
|------|---------|-------------|
| 首次访问（无 cookie） | 完整展示所有 10 个模块 | — |
| 已订阅 + 访问 > 3 次 | 日期头 + Tab + 内容卡片 + 导航 + 操作栏 | Coming Soon 折叠为一行「更多流派开发中 →」；订阅区隐藏或变「修改订阅偏好」 |

老用户主页核心只留 5 个模块：日期 → Tab → 内容 → 导航 → 操作栏。

## 2. 永久内容页 `/daily/yijing/[slug]`

与日课主页结构完全相同，区别：

- URL 是永久的（如 `/daily/yijing/qian-chujiu`）
- 有 canonical URL，SEO 友好
- 页面顶部增加面包屑：`首页 > 日课 > 易经卦序 > 乾卦·初九`
- `<head>` 中有独立的 title/description/og 标签
- 如果当天不是这一天的内容，显示提示：「这是第 N 天的内容。查看今日内容 →」

### 设计细节

- Tab 切换两个流派（同日课主页的 tab 样式）

- 顶部快速跳转栏：`[Day __ ▾]` `[卦名 ▾]` `[今天]` 三个控件
- 按卦/章分组，`<details>` 原生折叠（无 JS 也可用）
- 每条目录项：天数 + 爻位 + 爻辞摘要
- 当前天高亮（金色左边框 `border-l-2 border-gold` + 浅金底色）
- 已过天数正常显示，未来天数 `opacity-50`
- 点击 → 跳转到永久内容页
- 备选视图：「按月分组」（12 组，每组约 32 条），用 tab 或 toggle 切换

## 响应式断点

| 断点 | 布局变化 |
|------|---------|
| < 640px (mobile) | 单列，tab 全宽，卡片无 margin |
| 640-768px (tablet) | 同 mobile，稍宽 padding |
| > 768px (desktop) | max-width 768px 居中，同现有首页 |

日课是阅读体验，不需要复杂的响应式布局。保持单列、居中、最大宽度 768px，与现有页面一致。

## 无障碍 (a11y)

- Tab 切换：`role="tablist"` + `role="tab"` + `aria-selected`，键盘左右箭头切换
- 折叠区域：`<details>` 原生语义，屏幕阅读器自动支持
- 吉凶提醒区：`role="note"` + `aria-label="吉凶提醒"`，不依赖颜色传达信息（同时有 emoji + 文字标题）
- 倒计时：`aria-live="polite"` 每分钟播报一次（不逐秒，避免干扰）
- 所有交互元素 focus 可见：`focus-visible:ring-2 ring-gold/50`

---

## 暗色模式

所有新组件使用 `var(--theme-*)` CSS 变量，自动适配暗色模式：
- 卡片背景：`var(--theme-bg-card)`
- 文字：`var(--theme-text-primary/secondary/muted)`
- 边框：`var(--theme-border)`
- 金色保持不变（暗色下金色更突出）

## 状态矩阵（P2 补全）

### 页面级状态

| 状态 | 触发条件 | UI 表现 |
|------|---------|---------|
| Loading | 路由切换、首次加载 | 内容卡片区域显示骨架屏（Skeleton），日期头 + Tab 立即渲染（静态数据） |
| Error (API) | 内容接口 5xx / 超时 | 卡片区显示「内容加载失败，请刷新重试」+ 重试按钮，不影响导航和订阅区 |
| 404 (slug 不存在) | 访问无效 slug | 自定义 404：「这条日课还没有到来」+ 「查看今日内容 →」链接 + 目录入口 |
| 空状态 (EPOCH 前) | 日课未开播 | 重定向到倒计时页 `/daily`（Pre-launch） |

### 订阅流程状态

| 状态 | UI 表现 |
|------|---------|
| 默认（未订阅） | 邮箱输入框 + [免费订阅] 按钮 |
| 输入中 | 输入框 focus 态 `border-gold`，实时邮箱格式校验 |
| 提交中 | 按钮变 loading spinner，输入框 `disabled` |
| 提交成功 | 整个订阅区替换为 ✅「查看邮箱，点击验证链接即可完成订阅」 |
| 提交失败 | 输入框下方红色错误提示，按钮恢复可点击 |
| 已订阅用户 | 订阅区变为 ✅「已订阅，明天 8:00 见！」+ 「修改订阅偏好」文字链接 |

### 邮箱验证页 `/daily/verify`

| 状态 | UI 表现 |

### 吸底条状态

| 条件 | 行为 |
|------|------|
